# .gitignore Documentation

## Overview

**FILE**: `git/.gitignore`  
**PURPOSE**: Git version control exclusion rules for basketball statistics project

Specifies which files and directories Git should ignore (not track) to prevent committing temporary files, build artifacts, local configuration, and sensitive data to the repository.

## Rules Breakdown

### Python Compiled Files

```gitignore
# Byte-compiled / optimized / DLL files
__pycache__/
*.py[cod]
*$py.class
```

**Purpose**: Exclude Python bytecode and compiled files

**Files Ignored**:
- `__pycache__/` - Directory containing `.pyc` files
- `*.pyc` - Compiled Python files
- `*.pyo` - Optimized bytecode files
- `*.pyd` - Python DLL files (Windows)
- `*$py.class` - Jython class files

**Why Ignore**:
- Generated automatically by Python interpreter
- Platform and version specific
- Regenerated on each run
- Not portable across environments
- Bloat repository size

**Example Ignored Files**:
```
utils/__pycache__/
utils/__pycache__/accessing_data.cpython-313.pyc
main/__pycache__/main.cpython-313.pyc
testing/__pycache__/player_report.cpython-313.pyc
```

---

### VS Code Settings

```gitignore
# VS Code settings
.vscode/
```

**Purpose**: Exclude editor configuration directory

**Files Ignored**:
- `.vscode/settings.json` - Workspace settings
- `.vscode/launch.json` - Debug configurations
- `.vscode/extensions.json` - Extension recommendations
- `.vscode/tasks.json` - Task definitions

**Why Ignore**:
- Contains personal preferences
- May include local file paths
- Editor-specific (not universal)
- Each developer has own setup

**Trade-off**:
- **Current setup**: `.vscode/` ignored
- **Alternative**: Commit `.vscode/` for team consistency
- **Recommendation**: Remove this rule to share configurations

**Impact of Current Rule**:
- Team members must configure VS Code individually
- No shared debug configurations
- No shared workspace settings
- Inconsistent development environment

**Suggested Change**:
```gitignore
# Remove .vscode/ rule to enable sharing
# OR selectively ignore personal settings:
.vscode/settings.json    # Personal preferences
!.vscode/launch.json     # Shared debug configs
!.vscode/extensions.json # Recommended extensions
```

---

### Database Files

```gitignore
# Database / local data
Database/Data.json
```

**Purpose**: Exclude primary game statistics data file

**Files Ignored**:
- `Database/Data.json` - Main data storage

**Why Ignore**:
- Contains personal player data (privacy)
- May be large with many games
- Frequently modified (noise in commits)
- Each user may have different data

**Trade-off**:
- **Pros**: Privacy, clean commit history
- **Cons**: No data backup in repo, harder to share

**Alternatives**:

**Option 1 - Sample Data Only**:
```gitignore
Database/Data.json           # Ignore real data
!Database/Data.sample.json   # Include sample
```

**Option 2 - Anonymized Data**:
```bash
# Create anonymized version for repo
python scripts/anonymize_data.py Database/Data.json Database/Data.anonymized.json
```

**Option 3 - Commit Data**:
```gitignore
# Remove Database/Data.json rule
# Commit data to repo for backup/sharing
```

**Current Impact**:
- New clones have no data
- Must provide Data.json separately
- No version history of data changes
- No backup in Git

### Distribution and Packaging

```gitignore
# Distribution / packaging
build/
dist/
*.egg-info/
```

**Purpose**: Exclude Python package build artifacts

**Directories Ignored**:
- `build/` - Temporary build directory
- `dist/` - Distribution archives (wheels, tarballs)
- `*.egg-info/` - Package metadata directories

**Why Ignore**:
- Generated by setuptools/pip
- Temporary build artifacts
- Regenerated on each build
- Platform-specific binaries
- Unnecessary in version control

**Generated By**:
```bash
python setup.py build         # Creates build/
python setup.py sdist         # Creates dist/
pip install -e .              # Creates *.egg-info/
```

**Example Ignored Files**:
```
build/lib/utils/accessing_data.py
dist/basketball-stats-1.0.0.tar.gz
basketball_stats.egg-info/PKG-INFO
```

**Note**: Current project may not use packaging (no setup.py visible), but rule prevents future issues

### Operating System Files

```gitignore
# OS files

```

**Purpose**: Exclude OS-specific metadata files

**Files Ignored**:

**macOS**:
- `.DS_Store` - Finder metadata (folder view preferences)

**Windows**:
- `Thumbs.db` - Thumbnail cache

**Why Ignore**:
- OS-specific (not portable)
- Automatically generated
- No relevance to project
- Pollute repository
- Cause unnecessary merge conflicts

**Example Scenarios**:

# Windows user views images
screenshots/Thumbs.db        # Created automatically

## Common Issues and Solutions

### .vscode/ Already Tracked

**Problem**: `.vscode/` ignored but already committed

```bash
# Check if tracked
git ls-files .vscode/

# If tracked, remove from tracking
git rm -r --cached .vscode/
git commit -m "Remove .vscode/ from tracking"
```

**Result**: Files remain locally but removed from repo

---

### Database/Data.json Accidentally Committed

**Problem**: Data file in repository, need to remove

```bash
# Remove from tracking but keep local file
git rm --cached Database/Data.json
git commit -m "Remove Data.json from tracking"

# Verify ignored
git check-ignore Database/Data.json
```

---

### __pycache__ Already in Repo

**Problem**: Bytecode files committed before .gitignore created

```bash
# Find all __pycache__ directories
git ls-files | grep __pycache__

# Remove from tracking
find . -name __pycache__ -type d -exec git rm -r --cached {} +
git commit -m "Remove __pycache__ from tracking"
```

---

### Changes to Ignored Files Show Up

**Problem**: Modified ignored file appears in git status

**Cause**: File was tracked before ignore rule added

**Solution**:
```bash
# Remove from tracking
git rm --cached path/to/file

# Verify ignored
git check-ignore path/to/file
```

## Best Practices

### General Rules

**Do Ignore**:
- Generated files (can be recreated)
- Personal configuration (user-specific)
- Sensitive data (passwords, API keys)
- Build artifacts (temporary)
- OS metadata (irrelevant)

**Don't Ignore**:
- Source code (.py files)
- Documentation (.md files)
- Sample/test data
- Dependency lists (requirements.txt)
- Project configuration (shared settings)

## File Location Note

**Current Location**: `git/.gitignore`

**Standard Location**: `.gitignore` (project root)

**Issue**: Git looks for `.gitignore` in root and subdirectories

**Impact**: 
- `git/.gitignore` NOT automatically used by Git
- Must be at project root: `.gitignore`
- Or explicitly referenced in Git config

**Recommendation**:
```bash
# Move to standard location
mv git/.gitignore .gitignore

# Or create symlink
ln -s git/.gitignore .gitignore
```

**Verification**:
```bash
# Check if .gitignore is working
git check-ignore -v Database/Data.json

# If no output, .gitignore not found by Git
```

## Summary

`.gitignore` manages version control by:

**Excludes**:
- Python bytecode (`__pycache__/`, `*.pyc`)
- VS Code settings (`.vscode/`)
- Game data (`Database/Data.json`)
- Build artifacts (`build/`, `dist/`)
- OS files (`.DS_Store`, `Thumbs.db`)

**Benefits**:
- Clean repository (no generated files)
- Privacy protection (no personal data)
- Reduced merge conflicts
- Faster Git operations

**Issues**:
- File location incorrect (should be in root)
- `.vscode/` exclusion prevents sharing configs
- `Data.json` exclusion complicates setup

**Recommendations**:
1. Move file to project root (`.gitignore`)
2. Consider committing `.vscode/` for team consistency
3. Add sample data file for new developers
4. Add virtual environment and `.env` rules

**Critical**: Verify `.gitignore` is in correct location for Git to use it
