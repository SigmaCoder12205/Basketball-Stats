# .vscode Folder Documentation

## Overview

**FOLDER**: `.vscode/`
**PURPOSE**: VS Code workspace configuration directory for basketball statistics project

Contains workspace-specific settings, debugger configurations, and documentation for the development environment.

---

## Folder Structure

```
.vscode/
├── launch.json              # Debugger launch configurations
├── settings.json            # Workspace settings and preferences
└── doc/
    ├── launch.json_doc.md   # Launch configuration documentation
    └── settings.json_doc.md # Settings documentation
```

---

## Files Overview

### Configuration Files (Root Level)

#### `launch.json`
**Purpose**: Debug and run configurations for VS Code debugger

**Contents**:
- 4 launch configurations
- Python execution settings
- Working directory configuration
- Console output settings

**Key Configurations**:
1. **Run Current File** - Execute any open Python file
2. **Run Player Report** - Launch main GUI application
3. **Run Main** - Legacy main entry point
4. **Run AccessData Module** - Test data layer

**Usage**: Press F5 or select from debug dropdown

---

#### `settings.json`
**Purpose**: Workspace-specific VS Code preferences

**Contents**:
- Visual theme configuration (icons, colors)
- Spell checker custom dictionary
- Python testing framework setup

**Key Settings**:
- Material Icon Theme for file icons
- Evondev Dracula color scheme
- Pytest configuration
- Custom spell check words

**Usage**: Automatically applied when workspace opens

2. **Evondev Dracula Darker Contrast** (Evondev)

3. **Code Spell Checker** (Street Side Software)
   - Custom dictionary support
   - `cSpell.words` setting

4. **Python** (Microsoft)
   - Testing framework integration
   - `python.testing.*` settings

   - Launch configuration support

## Setup Guide

### New Developer Onboarding

1. **Clone Repository**
```bash
git clone <repository-url>
cd basketball-stats
```

2. **Open in VS Code**
```bash
code .
```

3. **Install Extensions**
- VS Code will prompt to install recommended extensions
- Or manually install from Extensions marketplace

4. **Install Python Dependencies**
```bash
pip install -r requirements.txt
# or
pip install pytest PyQt5
```

5. **Verify Configuration**
- Press F5 → Should see 4 launch configurations
- Open Testing sidebar → Should discover tests
- Check theme applied correctly

## Usage Workflows

### Running the Application

**Via Launch Configuration**:
1. Open debug dropdown (Ctrl+Shift+D)
2. Select "▶ Run Player Report"
3. Press F5 or click green play button
4. GUI launches with player statistics

**Via File Execution**:
1. Open `testing/player_report.py`
2. Select "▶ Run Current File"
3. Press F5

---

### Debugging Code

**Set Breakpoints**:
1. Click left margin in editor (red dot appears)
2. Select appropriate launch configuration
3. Press F5 to start debugging
4. Code pauses at breakpoint

**Debug Tools**:
- Step Over (F10)
- Step Into (F11)
- Step Out (Shift+F11)
- Continue (F5)
- Inspect variables in sidebar

---

### Running Tests

**Discover Tests**:
1. Open Testing sidebar (flask icon)
2. Tests discovered in configured directory
3. Tree view shows all test files and functions

**Run Tests**:
- Click "Run Test" above test function
- Click play button in Testing sidebar
- Right-click → Run Test

**Debug Tests**:
- Click "Debug Test" above test function
- Set breakpoints in test code
- Step through execution

## Common Issues and Solutions

### Extensions Not Installed

**Symptom**: Theme not applied, icons missing, testing broken

**Solution**:
1. Open Extensions (Ctrl+Shift+X)
2. Search for required extensions
3. Install and reload VS Code

**Check Installation**:
```bash
# View installed extensions
code --list-extensions
```

---

### Launch Configuration Not Working

**Symptom**: F5 does nothing, errors in debug console

**Solution**:
1. Verify Python interpreter selected
   - Ctrl+Shift+P → "Python: Select Interpreter"
2. Check file paths in `launch.json`
3. Verify `cwd` set correctly
4. Check dependencies installed

---

### Tests Not Discovered

**Symptom**: Testing sidebar empty, no tests found

**Solution**:
1. Verify pytest installed: `pip list | grep pytest`
2. Check test directory in `settings.json`:
   ```json
   "python.testing.pytestArgs": ["testing"]  // Correct path
   ```
3. Refresh tests: Testing sidebar → Refresh icon
4. Check test file naming: `test_*.py` or `*_test.py`

---

### Settings Not Applied

**Symptom**: Theme unchanged, spell check not working

**Solution**:
1. Reload VS Code: Ctrl+Shift+P → "Reload Window"
2. Check settings.json syntax (valid JSON)
3. Verify extension installed and enabled
4. Check for user settings override

```json
"cwd": "${workspaceFolder}"
```

## Customization Examples

### Add New Launch Configuration

```json
// launch.json
{
    "name": "▶ Run Team Report",
    "type": "python",
    "request": "launch",
    "program": "${workspaceFolder}/testing/team_report.py",
    "console": "integratedTerminal",
    "cwd": "${workspaceFolder}"
}
```

---

```json

    "AccessData",     // Add class names
```

---

### Configure Test Coverage

```json
// settings.json
"python.testing.pytestArgs": [
    "testing",
    "--cov=utils",           // Coverage for utils folder
    "--cov-report=html",     // HTML coverage report
    "-v"                     // Verbose output
]
```

## Security Considerations

### What to Commit

**Safe to Commit**:
- Launch configurations (generic paths)
- Workspace settings (no secrets)
- Documentation

**Do Not Commit**:
- API keys or tokens
- Personal file paths
- Private extension configs
- Machine-specific paths

### Sensitive Data

If settings contain sensitive data:
```gitignore
# .gitignore
.vscode/settings.json
```

Use environment variables instead:
```json
// launch.json
"env": {
    "API_KEY": "${env:MY_API_KEY}"  // From system environment
}
```
